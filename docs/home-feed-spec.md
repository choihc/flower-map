# 홈 피드 우선순위 규칙

## 목표

- 사용자가 앱 첫 진입 후 5초 안에 갈 만한 꽃 명소를 고를 수 있게 한다.

## 홈 섹션 순서

1. 현재 시즌 추천
2. 곧 끝나는 축제
3. 꽃 종류 바로가기
4. 지역별 추천

## 1. 현재 시즌 추천

노출 기준:

- `bloom_start_at <= today`
- `bloom_end_at >= today`
- `is_published = true`

정렬 우선순위:

1. `is_featured = true`
2. 사용자의 현재 지역 또는 최근 선택 지역과 일치
3. 인기 점수
4. 축제 진행 중

카드 구성:

- 대표 이미지
- 꽃 종류
- 시즌 배지
- 명소명
- 한 줄 설명
- `지도 보기`
- `길찾기`

비어 있을 때:

- "이번 주 추천 명소를 준비 중이에요" 문구와 함께 지역 추천으로 대체

## 2. 곧 끝나는 축제

노출 기준:

- `festival_end_at >= today`
- `is_published = true`

정렬 우선순위:

1. `festival_end_at` 오름차순
2. 사용자의 현재 지역 또는 최근 선택 지역과 일치
3. 인기 점수

카드 구성:

- 축제명
- 종료일까지 남은 일수
- 지역
- 연결된 대표 꽃 종류

## 3. 꽃 종류 바로가기

노출 기준:

- `flowers.is_active = true`

정렬 기준:

- `flowers.sort_order ASC`

초기 항목:

- 벚꽃
- 진달래
- 튤립
- 유채꽃

운영 원칙:

- 어드민에서 추가한 꽃 종류가 자동 반영된다.
- 비활성화된 꽃 종류는 홈과 지도 필터에서 제외한다.

## 4. 지역별 추천

초기 노출 지역:

- 서울/경기
- 부산/경남
- 전주/남도
- 제주

목적:

- 명확한 위치 탐색이 아직 없는 사용자를 지역 기반 탐색으로 유도한다.

## 빈 상태와 예외 처리

- 위치 권한이 없으면 최근 선택 지역 또는 전국 기준으로 노출한다.
- 현재 시즌 추천이 없으면 곧 끝나는 축제를 첫 섹션으로 올린다.
- 축제 데이터가 없는 명소도 현재 시즌 추천에는 포함할 수 있다.
