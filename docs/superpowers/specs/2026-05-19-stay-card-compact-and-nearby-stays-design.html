StayCard 컴팩트화 + 명소 → 호텔 연계 강화

2026-05-19 · feat/home-hocance-integration · 호캉스 Phase 2 후속
핵심 요약. StayCard를 가로형(112×112 이미지 + 우측 정보)으로 컴팩트화하면서 길찾기 버튼을 제거하고, 명소 상세 화면에 '주변 호텔' 섹션을 신설해 꽃 명소 ↔ 호텔 연계를 동선으로 연결한다. 카드 정체성은 예약 전환 1순위로 유지하되, 큐레이션 정책은 거리 0.7 + 평점 0.3으로 명소 옆 호텔 가중을 강화한다. PlaceCard 추상화·티켓/맛집 카테고리는 별도 PR.

1. 배경 / 목적

  • 호캉스 Phase 2(2026-05-12) 이후 StayCard가 약 400px로 커서 두 화면(홈 TOP5 / 호캉스 리스트) 모두 스크롤 부담. 길찾기 버튼은 실사용 비율이 낮다.
  • 꽃 명소 ↔ 호텔 연계는 카드 단위가 아닌 컨텍스트(섹션)에서 강화해야 한다. 명소 상세 화면에서 "이 명소 옆에서 자고 갈 곳" 동선이 단절돼 있다.
  • 향후 명소/티켓/맛집 카테고리 확장의 첫 적용 사례. 이번 PR은 호텔 케이스만 구현, PlaceCard 추상화는 후속.

// apps/mobile/src/features/spot/lib/findNearbyStays.ts
export type NearbyStay = { stay: Stay; distanceKm: number; score: number };
export type NearbyStaysResult = {
  stays: NearbyStay[];
  usedFallback: boolean;   // 30km 0건 → 60km 사용 시 true
};
export function findNearbyStays(
  spot: FlowerSpot,
  stays: Stay[],
  options?: FindNearbyOptions,
): NearbyStaysResult;
  • 좌표 결측 호텔 후보 제외
  • 1차: primaryRadiusKm 이내 후보 → limit 만큼 반환
  • 1차 0건일 때만 2차: fallbackRadiusKm 이내로 확장
  • score = 0.7 × (1 - d/fallbackRadiusKm) + 0.3 × normalizedRating
  • 동점 시 distanceKm 오름차순으로 결정적

4.4 UI 명세

  • 카드 boostBadge.label: 이 명소에서 ${formatDistance(distanceKm)}
  • 3개 아래 더보기 → 버튼 (/stays 라우팅). 후보가 limit 이하면 더보기 숨김

6. 테스트 전략

6.1 단위 (TDD)

  • findNearbyStays: 좌표 결측 제외 · 1차 후보 있을 때 fallback 미발동 · 1차 0건 + 2차 N건 → fallback 동작 · 60km도 0건이면 빈 배열 · 동점 시 distanceKm 오름차순 · limit 잘림
  • StayCard: 길찾기 버튼 미렌더 · 예약/카드 탭 콜백 호출 · boostBadge null이면 라인 미렌더 · 평점 없을 때 별 칩 미렌더 · 태그 3개 이상이어도 2개만
  • NearbyStaysSection: 0건 null 반환 · N건 카드 N개 + 더보기는 N > limit일 때만 · 거리 칩 라벨 1차/2차 분기

6.2 회귀 / 통합 / 수동

  • 호캉스 리스트 광고 슬롯(3개 간격) 위화감 점검
  • 홈 TOP5에 boost 있는/없는 카드 섞일 때 높이 동일
  • 명소 상세 (좌표 정상/외딴/좌표 결측 각각)

7. 후속 PR 후보

  • PlaceCard 추상화 — 좌이미지+우정보 패턴 일반화

8. 리스크 / 트레이드오프

  • 시각 임팩트 이미지 112×112로 축소되어 분위기 어필 약화. 비교 효율은 개선.
  • 라벨 외부화 호출부가 라벨 포맷 책임. 컨텍스트별로 다르므로 의도된 분산.
  • fallback 60km 도서·산악 명소에서 너무 멀 수도. 운영 데이터로 빈도 모니터링.
  • 가중치 차이 홈 0.5/0.5 vs 주변 0.7/0.3. 의도적 차이지만 추후 일관성 검토.
