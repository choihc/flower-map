# Admin Dashboard Redesign Design

## 1. Goal

기존 `apps/web`의 기능 흐름은 유지하면서, 화면 경험은 `토스 운영툴`에 가까운 절제형 대시보드 어드민으로 전면 재구성한다.

이번 리디자인의 목표는 다음 3가지다.

- 운영자가 첫 화면에서 현재 상태를 바로 파악할 수 있어야 한다.
- CRUD와 JSON import 작업이 더 구조적이고 덜 투박하게 느껴져야 한다.
- `shadcn/ui` 기반의 재사용 가능한 운영용 UI 레이어를 만들어 이후 확장 비용을 줄여야 한다.

이번 범위는 정보구조를 크게 바꾸기보다, 현재 구현된 꽃/명소/JSON import 흐름 위에 높은 완성도의 어드민 UI를 덧입히는 데 집중한다.

## 2. Visual Direction

### 2.1 Visual Thesis

밝은 회색 캔버스 위에 또렷한 숫자, 얇은 경계선, 절제된 파란 포인트를 사용하는 `세련된 토스 스타일 운영 대시보드`.

### 2.2 Mood

- 차갑지 않지만 절제된 인상
- 카드 더미가 아니라 정돈된 운영 패널 느낌
- 숫자와 상태가 먼저 읽히고, 장식은 뒤로 빠지는 구조
- 실무형 화면이지만 너무 무겁지 않고, 스타트업 SaaS 같은 산만함 없이 깔끔한 톤

### 2.3 Style Rules

- 배경: `#F6F7FB` 계열의 매우 밝은 회색
- 패널: 거의 흰색에 가까운 면과 얇은 `1px` 경계선 위주
- 타이포: 강한 대비, 짧은 문장, 넓은 여백
- 강조색: 토스 계열의 파란색을 CTA, 포커스, 링크에만 제한적으로 사용
- 그림자: 아주 약하게만 사용하거나 최소화하고, 위계는 간격과 선으로 해결
- 라운드: `12px` 내외의 단정한 반경

## 3. Architecture

### 3.1 Layout Model

전체 어드민은 다음 3단 구조를 사용한다.

1. 좌측 고정 사이드바
2. 상단 컨텍스트 바
3. 메인 콘텐츠 영역

사이드바는 항상 동일한 위치에서 탐색을 제공하고, 메인 영역은 `요약 -> 작업 -> 데이터` 순서로 읽히게 만든다.

### 3.2 Navigation

좌측 1단 사이드바를 사용한다.

- 대시보드
- 꽃 관리
- 명소 관리
- JSON 등록
- 설정

사이드바는 과도한 아이콘 세트보다 텍스트 중심으로 구성하고, 현재 페이지만 또렷하게 강조한다.

### 3.3 Top Context Bar

상단 바는 페이지 제목, 보조 설명, 우측 빠른 액션 영역으로 구성한다.

- 페이지별 한 줄 설명
- 최근 데이터 freshness 또는 운영 상태
- 주요 CTA 1개

헤더 자체를 마케팅 배너처럼 만들지 않고, 현재 화면의 목적을 빠르게 정렬하는 도구로 사용한다.

## 4. Page Design

### 4.1 Dashboard Home (`/`)

`/`는 더 이상 단순 리다이렉트가 아니라 실제 운영 대시보드 홈이 된다.

#### Sections

- KPI row
  - 총 꽃 수
  - 총 명소 수
  - draft 수
  - published 수
- Quick actions
  - 꽃 추가
  - 명소 추가
  - JSON 등록
- Recent updates
  - 최근 등록 명소
  - 최근 수정 명소
- Review queue
  - draft 대기 항목 목록

#### Layout Intent

첫 화면은 “많은 카드”보다 “큰 숫자, 짧은 라벨, 최근 작업 리스트”가 중심이 된다.

운영자는 진입 즉시

- 지금 얼마나 쌓여 있는지
- 무엇이 대기 중인지
- 어디로 바로 가야 하는지

를 읽을 수 있어야 한다.

### 4.2 Flowers (`/flowers`)

꽃 관리는 `목록 + 입력 패널` 구성이 기본이다.

#### Sections

- 상단 설명 + 꽃 추가 액션
- 좌측 꽃 목록
- 우측 생성/수정 패널

#### UX Rules

- 목록은 간결한 row 기반으로 표시
- 항목에는 이름, slug, 시즌, 활성화 상태만 우선 노출
- 선택 항목은 우측 폼에서 수정 가능한 구조로 확장 가능하게 설계
- 초기 구현은 생성 중심이어도 레이아웃은 수정 가능성을 염두에 둔다

### 4.3 Spots (`/spots`)

명소 관리는 운영에서 가장 자주 쓰는 화면이므로 가장 밀도 높게 구성한다.

#### Sections

- 상단 KPI strip
  - 총 명소
  - draft
  - published
  - 대표 노출 수(`is_featured`)
- 필터 bar
  - 꽃 종류
  - 상태
  - 지역
  - 검색
- 리스트 영역
- 우측 등록/수정 패널

#### UX Rules

- 리스트는 조용한 테이블 또는 dense list 스타일로 구현
- 각 행에는 이름, 꽃 종류, 지역, 상태, 최근 수정일, 대표 여부를 표시
- status는 작은 배지로 처리
- draft는 중성 톤, published는 파란 계열 또는 짙은 대비로 차별화
- 우측 패널 폼은 긴 폼을 한 덩어리로 보여주지 않고 구획을 나눈다

### 4.4 Import (`/spots/import`)

JSON import 화면은 “텍스트 편집기 + 결과 패널” 구조로 설계한다.

#### Sections

- 좌측: JSON 입력
- 우측: 검증 결과 / 생성 수 / 업데이트 수 / 저장 결과
- 하단 또는 측면: 최근 import 가이드

#### UX Rules

- 좌측 입력 영역은 편집기처럼 넉넉한 높이를 가진다
- 우측 결과 패널은 항상 보이는 요약 박스로 둔다
- 저장 액션은 `draft 저장`으로 명확히 표현한다
- 에러는 조용하지만 명확한 경고 스타일로 보여준다

## 5. Component System

### 5.1 shadcn/ui Scope

우선 도입 대상은 아래 컴포넌트들이다.

- `Button`
- `Card` 또는 panel wrapper
- `Input`
- `Textarea`
- `Select`
- `Badge`
- `Table`
- `Separator`
- `Sheet` 또는 `Dialog`

단, `shadcn/ui`를 가져오더라도 기본 스타일을 그대로 두지 않고 이 프로젝트 시각 톤에 맞게 토큰을 다시 맞춘다.

### 5.2 Custom Admin Components

추가로 아래 공통 컴포넌트를 만든다.

- `AdminSidebar`
- `AdminTopbar`
- `MetricCard`
- `SectionHeader`
- `DataListTable`
- `StatusBadge`
- `QuickActionPanel`
- `FormSection`

이 컴포넌트들은 단순 래퍼가 아니라, 어드민 전반의 간격/타이포/상태 표현을 통일하는 역할을 맡는다.

## 6. Typography and Spacing

### 6.1 Typography

- 숫자는 페이지에서 가장 강하게 보이도록 큰 크기와 높은 대비를 사용
- 제목은 짧고 명확하게 유지
- 설명문은 1문장 이내로 제한
- 본문은 읽기보다 훑기 용도로 설계

### 6.2 Spacing

- 화면 전체는 넉넉한 outer padding
- 섹션 간 여백은 분명하게
- 같은 패널 안에서는 촘촘하지만 답답하지 않은 간격 사용

핵심은 “커 보이는 카드”보다 “정확한 정렬과 리듬감”이다.

## 7. Form Design

### 7.1 Flower Form

꽃 폼은 짧고 빠르게 입력 가능한 구조를 유지하되, 시각적으로는 아래 순서를 따른다.

- 기본 정보
- 시즌 정보
- 표시 설정

### 7.2 Spot Form

명소 폼은 다음 섹션으로 나눈다.

- 기본 정보
- 위치 정보
- 개화/축제 일정
- 이미지/메모
- 공개 상태

각 섹션은 제목과 짧은 설명을 갖고, 필드 간 경계가 명확해야 한다.

### 7.3 Validation UX

- 에러는 필드 근처와 상단 요약 중 한 곳 이상에서 보여준다
- 성공 상태는 과한 초록 알림 대신, 조용한 완료 문구로 처리한다
- 저장 버튼은 항상 화면 맥락을 반영한 문구를 사용한다
  - 예: `꽃 저장`, `명소 저장`, `draft 저장`

## 8. Data Presentation

### 8.1 KPI Presentation

KPI는 장식적 위젯이 아니라 운영 숫자다.

- 값이 가장 크고
- 라벨은 짧고
- 보조 설명은 선택적으로만 사용

### 8.2 List Presentation

목록은 카드 그리드가 아니라 row 기반으로 정리한다.

- 스캔성이 우선
- 선택과 수정이 빠르게 이어질 수 있어야 함
- 배경색보다 상태/텍스트/정렬로 정보 차이를 만든다

## 9. Motion

모션은 작지만 존재감 있게 사용한다.

- 페이지 진입 시 KPI와 주요 패널이 짧게 나타나는 reveal
- 사이드바 active indicator의 부드러운 전환
- hover 시 row 강조 정도의 얕은 인터랙션

애니메이션은 운영 툴 특성상 빠르고 얕게 유지하며, 장식적 전환은 피한다.

## 10. Accessibility

- 명도 대비는 충분히 확보한다
- 상태 배지는 색만으로 구분하지 않는다
- 폼과 테이블은 키보드 접근성을 유지한다
- 모바일에서도 깨지지 않지만, 우선순위는 데스크톱 운영 화면에 둔다

## 11. Implementation Scope

이번 리디자인에서 구현할 범위:

- `shadcn/ui` 도입 및 토큰 조정
- 글로벌 레이아웃 재설계
- `/` 대시보드 홈 구성
- `/flowers`, `/spots`, `/spots/import` 화면 재구성
- 공통 어드민 UI 컴포넌트 정리

이번 리디자인에서 제외:

- 새로운 데이터 모델 추가
- 권한 체계 확장
- 대규모 차트 시스템
- 복잡한 설정 페이지 구현

## 12. Success Criteria

- 첫 화면이 “투박한 폼 모음”이 아니라 운영 대시보드처럼 느껴질 것
- CRUD와 import가 더 구조적이고 세련된 작업 흐름으로 보일 것
- 토스 스타일을 참조하되, 복제처럼 보이지 않고 서비스에 맞는 절제된 톤을 가질 것
- 향후 기능 추가 시 같은 시각 언어를 유지할 수 있는 공통 UI 기반이 생길 것
