홈 로딩 스켈레톤 스펙

apps/mobile 2026-06-05 SDD · 결과물 명세
대상: Expo 모바일 앱 · web · toss-mini · 공유 패키지와 무관

핵심 요약

홈 화면의 섹션별 제각각 로딩(pop-in)을 제거하고, 핵심 데이터 4종이 모두 준비될 때까지 하나의 통합 스켈레톤을 보여준 뒤 실제 홈을 한 번에 노출한다. 일부 쿼리가 실패하거나 느려도 안전 타임아웃(5초)으로 갇히지 않는다. 애니메이션은 신규 의존성 없이 react-native-reanimated 펄스로 구현하며, 공용 SkeletonBox를 업그레이드해 8개 화면의 로딩을 일관되게 개선한다.

1. 개요

홈은 서로 다른 시점에 끝나는 여러 데이터 쿼리에 의존한다. 각 섹션이 개별 로딩 상태를 노출하는 대신, useHomeReady가 핵심 쿼리를 일원화해 "노출 준비됨"을 판정하고, 준비 전에는 HomeSkeleton(차분한 단순 블록)만 보여준다. SkeletonBox는 펄스 애니메이션을 가진 공용 프리미티브다.

2. 기능 요구사항 FR

  • FR-1 핵심 쿼리 일원화 — useHomeReady는 핵심 4종을 섹션과 동일 키/함수/staleTime으로 실행. 섹션이 같은 키를 재사용 → 캐시 공유 → 준비 후 섹션 자체 스켈레톤 미발동.
    데이터쿼리 키조회 함수staleTime
    명소 목록spotKeys.allgetPublishedSpots기본
    홈 큐레이션homeCurationKeys.activegetActiveHomeCurationSlots30분
    꽃 명소 TOP10spotKeys.top(10)getTopSpots(10)30분
    호캉스 호텔stayKeys.allgetPublishedStays기본
  • FR-2 준비 판정 — ready는 4쿼리가 모두 settled(status !== 'pending', 성공 또는 에러)일 때 true. 실패 영역은 각 섹션의 빈/에러 상태가 처리.
  • FR-3 안전 타임아웃 — settled 미완이라도 HOME_SKELETON_TIMEOUT_MS = 5000ms 경과 시 ready 강제 true.
  • FR-4 통합 노출 — ready=false 동안 실제 섹션 미렌더, HomeSkeleton만. ready=true 시 전체 동시 노출.
  • FR-5 캐시 적중 무플래시 — 재방문 등 4쿼리 이미 settled면 최초 렌더에서 즉시 ready=true → 깜빡임 없음.
  • FR-6 스켈레톤 레이아웃 — 히어로 블록 1 + 동일 너비 카드 3(가로) + 와이드 카드 2(세로). 실제 섹션과 1:1 대응 아님; 정돈된 단일 면이 목적.

3. 비기능 요구사항 NFR

  • NFR-1 무신규 의존성 — 펄스는 설치본 react-native-reanimated로. expo-linear-gradient 등 추가 금지.
  • NFR-2 테스트 안전성 — 애니메이션은 순수 시각 효과. vitest + react-native mock에서 렌더를 깨지 않으며 정적 렌더 허용. 기존 getAllByTestId('skeleton-box') 테스트 무변경 통과.
  • NFR-3 섹션 비침습 — 섹션 props·로직·테스트 무변경. TopSpotsSection 내부 isLoading 분기 유지(안전망).
  • NFR-4 앱 격리 — 변경은 apps/mobile/에 국한.
  • NFR-5 성능 — 추가 쿼리는 섹션과 키 동일 → 중복 네트워크 없음(디둑·staleTime).

5. 구성 요소별 책임과 계약

5.1 SkeletonBox (공용, 수정)

type SkeletonBoxProps = {
  height: number;
  borderRadius?: number;   // 기본 16
  width?: string | number; // 기본 '100%'
  testID?: string;
};
export function SkeletonBox(props: SkeletonBoxProps): JSX.Element;

둥근 플레이스홀더 + 부드러운 펄스. props·배경색(#F2EDE6)·testID·외형 계약 유지(애니메이션만 가산). 동작 불가 환경에선 정적 degrade.

5.2 HomeSkeleton (신규)

export function HomeSkeleton(): JSX.Element;

props 없음. SkeletonBox로 FR-6 레이아웃 구성. 루트 testID="home-skeleton". ScreenShell 내부 배치 전제(자체 패딩/스크롤 미관리).

5.3 useHomeReady (신규)

export const HOME_SKELETON_TIMEOUT_MS = 5000;
export function useHomeReady(options?: { timeoutMs?: number }): { ready: boolean };

FR-1의 4쿼리 실행. ready = (4쿼리 모두 status !== 'pending') 또는 (timeoutMs 경과). 타임아웃은 마운트 시 1회 설정, 언마운트 시 해제.

5.4 HomeScreen (수정)

최상단 const { ready } = useHomeReady(); → !ready면 <ScreenShell><HomeSkeleton/></ScreenShell>, 아니면 기존 렌더 유지. 기존 spots.all·curation.active 조회는 그대로(데이터 사용), 훅과 캐시 공유.

명시적 가정

  • 조회 함수·쿼리 키는 현행대로 존재하며 섹션이 동일 키 사용(캐시 공유 전제).
  • React Query v5의 status/isPending 의미를 따른다(에러 시 pending 해제).
  • reanimated는 디바이스/시뮬레이터에서 동작, 테스트 mock에선 정적일 수 있음(허용).
