핵심 요약

  • 각 호텔은 두 provider 목적지를 데이터로 보유: trip.com 전체 URL(tripcom_booking_url) + Agoda hid(agoda_hotel_id).
  • 두 provider는 항상 함께 제공된다. 직링크 데이터가 없으면 각자 검색 fallback으로 동작하므로 진입점은 호텔별 데이터 유무와 무관.
  • 노출: 콤팩트(Top5·리스트·근처·hero) → provider 선택 바텀시트, 상세 예약 섹션 → 두 버튼 동시.
  • DB 마이그레이션 불필요 — agoda_hotel_id 컬럼은 trip.com 전환 시 보존됨. 코드 배선만 복원.
  • NFR-5 개정: trip.com은 전역 ID 비의존, Agoda는 의도적으로 EXPO_PUBLIC_AGODA_CID 전역 cid 사용.
  • 두 앱 격리: 토스 미니앱·공유 패키지 무영향.

1. 개요

호캉스(stays)의 "예약하기" 액션이 사용하는 제휴 링크 기능. 각 호텔은 두 예약 provider 목적지를 데이터로 보유한다.

  • trip.com — 호텔별 제휴 예약 URL 전체(tripcom_booking_url)를 그대로 오픈. 없으면 kr.trip.com 호텔명 검색.
  • Agoda — 호텔별 hid(agoda_hotel_id)와 전역 cid(EXPO_PUBLIC_AGODA_CID)로 Partner Search 호텔 페이지 URL을 조립해 오픈. hid 없으면 호텔명 검색.

도메인 책임: 호텔 → (provider별) 예약 목적지 URL 결정(직링크 우선, 없으면 검색) → 외부 브라우저 오픈.

2. 기능 요구사항 (FR)

FR-1 trip.com 호텔별 예약 URL
각 stay는 선택 필드 tripcom_booking_url(전체 URL). 없으면 null.
FR-1A Agoda 호텔별 hid
각 stay는 선택 필드 agoda_hotel_id(hid). 없으면 null.
FR-2 모바일 오픈 규칙
trip.com: URL이 http(s)면 그대로 / 아니면 검색.
Agoda: hid 있으면 직링크(cid+hid) / 없으면 검색(cid+hname).
FR-3 검색/직링크 URL 빌더
buildTripcomHotelSearchUrl · buildAgodaHotelSearchUrl · buildAgodaHotelDeepLink.
FR-4 오픈 실패 처리
Linking.openURL 실패 시 검색 키워드 클립보드 복사 + 안내 Alert (provider 공통).
FR-5 어드민 입력 — 카드 2개
trip.com URL 카드(전체 URL) + Agoda 호텔 ID 카드(전체 URL→hid 추출 또는 숫자 직접). 빈 입력 = 검색 fallback. 상태 배지·결과 메시지.
FR-6 JSON import
tripcom_booking_url·agoda_hotel_id 각각 옵션 필드(단건·일괄).
FR-7 예약 진입점
콤팩트(Top5·리스트·근처 카드·상세 hero) → 선택 바텀시트. 상세 예약 섹션 → 두 버튼 동시.
FR-8 상세 문구
검색어 라벨 일반화 검색어: "<query>". 버튼 "trip.com에서 예약하기 →" / "Agoda에서 예약하기 →". hero "예약하기 →" 유지(시트).
FR-9 provider 선택 바텀시트
공용 BookingProviderSheet(RN Modal). 두 행 항상 노출, 행 선택 시 해당 오픈 함수 호출 후 닫기.

3. 비기능 요구사항 (NFR)

NFR-1 URL 스킴 안전성
trip.com URL은 http(s)만(저장·import·모바일 방어선). Agoda hid는 숫자만 — 알려진 agoda.com 템플릿에만 끼워짐.
NFR-2 앱 업데이트 없는 목적지 갱신
trip.com: DB URL로 완전 제어. Agoda: 호텔별 hid는 DB로 제어(재배포 불필요), 단 URL 템플릿·전역 cid는 바이너리에 인라인.
NFR-3 두 앱 격리
apps/web·apps/mobile에만 존재. 토스 미니앱·공유 패키지 무영향.
NFR-4 쿼리 견고성
모바일은 select('*'). 매퍼는 없는 필드를 안전하게 null 처리.
NFR-5 전역 제휴 ID — provider별
trip.com 경로는 전역 ID 비의존(추적은 저장 URL에 포함). Agoda 경로는 의도적으로 EXPO_PUBLIC_AGODA_CID 사용. 미설정 시 cid 없이 진입(추적 불가, 허용 degrade).
2026-06-08 개정 메모: 최초 스펙은 trip.com 단일 provider 전제로 NFR-5를 "전역 ID 비의존"으로 두었다. Agoda 승인에 따라 두 provider 공존으로 갱신, NFR-5를 provider별로 재정의. agoda_hotel_id 컬럼은 보존되어 있어 마이그레이션 불필요.

5. 구성 요소 책임 · 계약

5.1 모바일 — affiliateHotel.ts

resolveBookingQuery(name, override): string
buildTripcomHotelSearchUrl(query): string
buildAgodaHotelSearchUrl(query): string      // cid+hname
buildAgodaHotelDeepLink(hotelId): string     // cid+hid
openTripcomHotel({ name, queryOverride, tripcomBookingUrl }): Promise<void>
openAgodaHotel({ name, queryOverride, agodaHotelId }): Promise<void>

Agoda 빌더는 process.env.EXPO_PUBLIC_AGODA_CID에서 cid를 읽는다. trip.com 경로는 전역 ID를 읽지 않는다. 두 오픈 함수는 실패 시 동일 클립보드+Alert.

5.2 모바일 — BookingProviderSheet

props: { stay: Stay | null; onClose: () => void }   // stay=null → 닫힘

두 행 항상 노출. 선택 시 해당 오픈 함수 호출 후 onClose(). 소비 화면은 useState<Stay|null> 보유, 화면당 인스턴스 1개.

5.3 타입 / 매퍼

Stay.agodaHotelId·StayRow.agoda_hotel_id 재추가, toStay에 agodaHotelId: row.agoda_hotel_id ?? null.

5.5 웹 — agodaHidParser

parseAgodaHotelId(input): 숫자면 그대로, agoda.com URL이면 hid 추출, 그 외 null.

5.6 웹 — 액션/데이터

updateStayAgodaHotelIdAction: trim 후 빈값→null, 비어있으면 파서로 hid 추출(실패 시 Error). buildStayWriteInput에 두 필드 emptyToNull.

5.7 웹 — import 스키마

tripcom_booking_url: httpsOnlyUrlSchema.nullable().optional(),
agoda_hotel_id: z.string().trim().regex(/^\d+$/).nullable().optional(),

5.8 웹 — 어드민 카드 2개

"trip.com 예약 URL" 카드 + "Agoda 호텔 ID" 카드. 각각 입력·저장·상태 배지·결과 메시지.

7. 외부 의존성

  • trip.com 검색: kr.trip.com/hotels/list?keyword=&locale=ko-KR&curr=KRW
  • Agoda Partner Search: agoda.com/partners/partnersearch.aspx?cid=&hl=ko-kr&hid=(직링크) / &hname=(검색)
  • Agoda cid: EXPO_PUBLIC_AGODA_CID env (빌드 인라인)
  • react-native Linking·Alert·Modal, expo-clipboard(선택), supabase, zod

8. 명시적 가정

  • tripcom_booking_url에는 제휴 추적 파라미터가 이미 포함(앱은 덧붙이지 않음).
  • EXPO_PUBLIC_AGODA_CID는 승인된 cid. 미설정 시 추적 불가(허용 degrade).
  • trip.com 검색 fallback은 추적 없음(수익 미인정, 의도된 허용).
  • trip.com keyword, Agoda hid/hname 파라미터 유효.
  • agoda_hotel_id 컬럼이 DB에 이미 존재 — 마이그레이션 불필요.
