문서 종류: 스펙(결과물 명세) · 도메인: 호캉스(stays) 예약 제휴 링크 · 최종 수정: 2026-06-02
연관 플랜: docs/plans/2026-06-02-agoda-to-tripcom-affiliate.md

핵심 요약

각 호텔은 자신의 trip.com 제휴 예약 URL 전체를 데이터로 보유하고, 모바일 예약 버튼은 그 URL을 그대로 연다. 없으면 trip.com 호텔명 검색으로 fallback.

예약 목적지 URL은 DB에 저장된 값이므로(앱이 조립하지 않음), 신버전 앱은 앱 업데이트 없이 DB 변경만으로 목적지를 갱신한다. 데이터 입력 경로는 ① 웹 어드민 stay 상세 폼, ② JSON import(단건/일괄). 영향 범위는 apps/web apps/mobile뿐이며 토스 미니앱·공유 패키지는 무영향.

1. 개요

호캉스(stays) 도메인의 "예약하기" 액션이 사용하는 제휴 링크 기능. 각 호텔(stay)은 자신의 trip.com 제휴 예약 URL 전체를 데이터로 보유하며, 모바일 앱의 예약 버튼은 이 URL로 외부 브라우저를 연다. URL이 없는 호텔은 trip.com 호텔명 검색 페이지로 이동한다.

도메인 책임: 호텔 → 예약 목적지 URL 결정(저장된 제휴 URL 우선, 없으면 검색) → 외부 브라우저 오픈. 예약 목적지 URL은 DB에 저장된 값이며 앱은 가공 없이 그대로 연다(전역 제휴 ID로 조립하지 않음). 데이터 입력 경로는 웹 어드민 폼과 JSON import 두 가지.

2. 기능 요구사항 (FR)

FR-1
호텔별 예약 URL 보유 — 각 stay는 선택적 tripcom_booking_url(전체 trip.com 제휴 URL)을 가진다. 없으면 null.
FR-2
모바일 오픈 규칙 — URL이 있으면 그 URL을 그대로 오픈, 없으면 resolveBookingQuery(name, override) 키워드로 trip.com 호텔 검색 URL 오픈.
FR-3
검색 URL 빌더 — buildTripcomHotelSearchUrl(query) → https://kr.trip.com/hotels/list?keyword=<enc>&locale=ko-KR&curr=KRW
FR-4
오픈 실패 처리 — Linking.openURL 실패 시 키워드를 클립보드 복사 + 호텔명 안내 Alert(기존 동작 보존).
FR-5
어드민 입력 — stay 상세에서 전체 URL 붙여넣어 저장/수정/제거. 빈 입력 → null. 상태 배지(직링크/검색 fallback), 결과 메시지.
FR-6
JSON import — stay import JSON에 tripcom_booking_url 옵션 필드(단건·일괄).
FR-7
예약 버튼 노출 지점 — 홈 Top5 · 호캉스 리스트 · 호캉스 상세 · 명소 "근처 호캉스" 4곳에서 공통 오픈 함수 호출.
FR-8
상세화면 문구 — "trip.com 검색어: …", "trip.com에서 호텔 예약하기 →". hero 주 버튼 "예약하기 →"는 유지.

3. 비기능 요구사항 (NFR)

NFR-1
URL 스킴 안전성 — 저장·import 시 http(s) 스킴만 허용(XSS 방어). 특정 도메인(trip.com)은 강제하지 않음(추적/리다이렉트 도메인 허용).
NFR-2
앱 업데이트 없는 목적지 갱신 — 목적지는 DB 저장 URL이므로 신버전 앱은 재배포 없이 DB 변경만으로 목적지 변경 반영.
NFR-3
두 앱 격리 — apps/web·apps/mobile에만 존재. 토스 미니앱·공유 패키지 무영향.
NFR-4
쿼리 견고성 — 모바일 select('*') 유지, 매퍼는 없는 필드를 null로 안전 처리.
NFR-5
전역 제휴 ID 비의존 — CID류 환경변수 불필요. 검색 fallback은 제휴 추적 없음(수익 미인정 허용).

5. 구성 요소별 책임과 계약

5.1 모바일 — affiliateHotel.ts

export function resolveBookingQuery(name: string, override: string | null): string;
export function buildTripcomHotelSearchUrl(query: string): string;
export function openTripcomHotel(opts: {
  name: string;
  queryOverride: string | null;
  tripcomBookingUrl: string | null;
}): Promise<void>;
  • openTripcomHotel: tripcomBookingUrl trim 후 길이>0이면 그 값, 아니면 검색 URL 오픈. 전역 제휴 ID 미사용.

5.2 모바일 — 타입 / 매퍼

  • Stay.tripcomBookingUrl: string | null, StayRow.tripcom_booking_url: string | null
  • toStay(row) → tripcomBookingUrl: row.tripcom_booking_url ?? null

5.3 모바일 — 호출 지점 (4곳 공통)

openTripcomHotel({ name: stay.name, queryOverride: stay.bookingQueryOverride, tripcomBookingUrl: stay.tripcomBookingUrl });

5.4 웹 — 서버 액션 / 데이터

updateStayTripcomUrlAction(id: string, rawUrl: string): Promise<void>
updateStayTripcomUrl(client, id: string, url: string | null): Promise<void>
  • trim → 빈 문자열 null; 값 있으면 http(s) 검증 실패 시 Error throw; revalidatePath 갱신.

5.5 웹 — import 스키마

tripcom_booking_url: httpsOnlyUrlSchema.nullable().optional(),

buildStayWriteInput은 emptyToNull로 정규화.

5.6 웹 — 어드민 폼 카드

카드 제목 "trip.com 예약 URL". 입력 1개(전체 URL) + 저장 + 상태 배지 + 결과 메시지.

7. 외부 의존성

  • trip.com 한국어 호텔 검색: https://kr.trip.com/hotels/list?keyword=<query>&locale=ko-KR&curr=KRW
  • trip.com 제휴 URL: 사용자가 어드민/import로 제공, 앱은 가공 없이 오픈.
  • react-native Linking/Alert, expo-clipboard(선택), @supabase/supabase-js.
확인 대상: kr.trip.com keyword 파라미터는 2026-06-02 기준 호텔 검색 UI 진입 확인. 빌더 함수에 캡슐화되어 구현 단계에서 손쉽게 재검증·조정.

8. 명시적 가정

  • 입력 tripcom_booking_url에는 제휴 추적 파라미터가 이미 포함(앱은 덧붙이지 않음).
  • 검색 fallback은 제휴 추적 없음(수익 미인정) — 의도된 허용 동작(NFR-5).
  • kr.trip.com 검색 keyword 파라미터 유효(§7 재검증 대상).
  • 외부 오픈은 기기 기본 브라우저/앱 처리에 위임.
