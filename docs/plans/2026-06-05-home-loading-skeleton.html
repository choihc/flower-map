홈 로딩 스켈레톤 — 구현 플랜

2026-06-05 · 대상 앱: apps/mobile (Expo) · 스펙: docs/specs/2026-06-05-home-loading-skeleton-spec.md

목표 — 홈 화면의 흩어진 섹션별 로딩 표시를 없애고, 핵심 데이터가 모두 준비될 때까지 단일 통합 스켈레톤을 보여준 뒤 실제 홈을 한 번에 노출한다.

아키텍처 — useHomeReady 훅이 홈 핵심 4쿼리를 섹션과 동일한 키로 관찰해 "모두 settled OR 타임아웃" 게이트를 만든다. 게이트 전엔 HomeSkeleton만, 후엔 기존 섹션을 그대로 렌더. 섹션이 같은 키를 재사용하므로 React Query 캐시가 공유되어 게이트 통과 직후 자체 스켈레톤 없이 즉시 렌더된다.

기술 — React Native(Expo SDK 55), React Query v5, react-native-reanimated 4.2.x(펄스), Vitest 3 + jsdom(테스트). 신규 의존성 없음.

Task 2 — SkeletonBox 펄스 애니메이션

스펙 5.1 · FR-6 NFR-1 NFR-2

계약 보존: props 시그니처·배경색 #F2EDE6·testID·외형은 그대로, 펄스(opacity breathing)만 가산. 기존 사용처 8곳과 테스트(top-spots-skeleton, skeleton-box)는 무변경 통과.

핵심 구현: useSharedValue(1) → useEffect에서 withRepeat(withTiming(0.45, …), -1, true) → useAnimatedStyle({opacity}) 를 Animated.View에 적용.

const opacity = useSharedValue(1);
useEffect(() => {
  opacity.value = withRepeat(
    withTiming(0.45, { duration: 800, easing: Easing.inOut(Easing.ease) }),
    -1, true,
  );
}, [opacity]);
const animatedStyle = useAnimatedStyle(() => ({ opacity: opacity.value }));

회귀 가드: TopSpotsSection.test / SearchScreen.test 실행해 스켈레톤 testID 조회가 그대로 잡히는지 확인(Step 5).

Task 3 — useHomeReady 준비 판정 훅

스펙 5.3 · FR-1 FR-2 FR-3 FR-5 NFR-5

핵심 4쿼리를 섹션과 동일 키/함수/staleTime으로 useQuery 실행. ready = 모두 settled(status !== 'pending') OR timeoutMs 경과. 기본 타임아웃 HOME_SKELETON_TIMEOUT_MS = 5000.

const allSettled = [spots, curation, topSpots, stays]
  .every((q) => q.status !== 'pending');
const [timedOut, setTimedOut] = useState(false);
useEffect(() => {
  const id = setTimeout(() => setTimedOut(true), timeoutMs);
  return () => clearTimeout(id);
}, [timeoutMs]);
return { ready: allSettled || timedOut };

테스트(5) — 전부 pending→false / 전부 success→true(무플래시) / 일부 error라도 settled→true / 하나라도 pending→false / 타임아웃 경과→강제 true. useQuery를 queryKey 기준 mockImplementation으로 제어(호출 순서 비의존). 파일은 .test.tsx(jsdom).

Task 5 — HomeScreen 게이트 통합

스펙 5.4 · FR-4 NFR-3

Rules of Hooks 준수: 모든 훅(useHomeReady, 기존 2×useQuery, 2×useMemo)은 매 렌더 무조건 호출되고, 스켈레톤 early return은 모든 훅 호출 이후에 둔다.
const { ready } = useHomeReady();
// …기존 useQuery/useMemo…
if (!ready) {
  return (<ScreenShell><HomeSkeleton /></ScreenShell>);
}
return (<ScreenShell>{/* 기존 섹션 전체 */}</ScreenShell>);

테스트 — 기존 3개는 useHomeReady를 ready:true 기본 mock으로 보존. 신규 2개: ready=false→home-skeleton 노출 & top-spots-section 없음 / ready=true→섹션 노출 & 스켈레톤 없음.
