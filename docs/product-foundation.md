# 꽃 어디 제품 설계 초안

## 1. 제품 포지셔닝

`꽃 어디`는 "지금 가장 예쁘게 피어 있는 꽃 스팟을 빠르게 찾는" 지도 기반 서비스입니다.

핵심 사용자:

- 주말 나들이 장소를 찾는 가족/연인
- 사진 찍기 좋은 계절 명소를 찾는 SNS 사용자
- 지역별 축제와 개화 시기를 한눈에 보고 싶은 여행객

## 2. MVP 범위

### 사용자 앱

- 현재 위치 또는 선택 지역 기준 꽃 명소 조회
- 꽃 종류 필터
- 명소 상세 카드/상세 페이지
- 축제 기간, 주소, 설명, 주차, 입장료 정보 제공
- 네이버 지도/카카오맵/카카오내비 등 외부 이동
- 즐겨찾기 저장

### 어드민

- 공공데이터 기반 명소 데이터 검수/수정
- 수동 신규 명소 추가
- 노출 여부 관리

### MVP 제외

- 사용자 사진 업로드
- 리뷰/커뮤니티
- 개화 상태 실시간 공유
- 통계 대시보드 고도화

## 3. 앱 정보 구조

### 탭 구조 제안

1. 홈
2. 지도
3. 즐겨찾기
4. 내 정보

### 핵심 흐름

#### 홈

- 현재 날짜 기준 개화 시즌 카드
- 종료되지 않은 가까운 축제 카드
- 인기 지역/추천 지역 카드
- 꽃 종류별 빠른 진입
- 지도 바로가기 CTA

#### 지도

- 네이버 지도 전체 화면
- 하단 플로팅 필터 칩
- 선택된 명소 요약 카드
- 리스트/지도 전환

#### 상세

- 대표 이미지
- 개화 상태 배지
- 축제 기간
- 주소 / 운영 정보 / 주차 / 입장료
- 길찾기 버튼
- 즐겨찾기 버튼

#### 즐겨찾기

- 저장한 명소 리스트
- 시즌 임박 알림 준비 상태 표시

## 7. 운영 설계 원칙

- 꽃 종류는 앱 코드에 하드코딩하지 않고 `flowers` 테이블 기준으로 관리한다.
- 어드민에서 꽃 종류를 추가/비활성화/정렬할 수 있게 설계한다.
- 명소는 하나의 대표 꽃 종류를 우선 연결하되, 추후 다중 연결이 가능하도록 확장 여지를 둔다.
- 홈 추천은 "현재 날짜 기준 개화 시즌"과 "종료되지 않은 행사"를 우선 노출한다.

## 8. UX 원칙

- 첫 화면에서 "지금 어디가 예쁜지"를 바로 보여준다.
- 사용자는 검색보다 탐색을 더 많이 할 가능성이 높다.
- 지도는 복잡하면 안 되고, 꽃 종류와 시즌 맥락이 잘 보여야 한다.
- 상세 페이지는 방문 결정을 도와주는 정보 순서로 구성한다.
- 정보형 구조를 유지하되, 카드/컬러/이미지 톤은 감성적으로 가져간다.

## 9. 빌드 전략 제안

### 1단계

- React Native + Expo 앱 우선 제작
- Supabase 스키마 정의
- 최소 어드민 CRUD 구축

### 2단계

- 공공데이터 수집 자동화
- 위치 기반 추천
- 시즌 큐레이션

### 3단계

- 사용자 업로드와 커뮤니티 기능

## 10. 현재 합의된 방향

- 홈 첫 화면 중심: 추천 카드 우선
- 디자인 성격: 정보형 구조 + 감성적인 톤
- 초기 꽃 종류: 벚꽃, 진달래, 튤립, 유채꽃
- 꽃 종류 확장 방식: 어드민에서 추가/수정 가능
- 모바일 스택: React Native + Expo
