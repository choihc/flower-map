# 모바일 스택 결정

## 결정

`꽃 어디` 모바일 앱의 MVP 스택은 `React Native + Expo`로 간다.

## 이유

- 웹 프론트엔드 경험을 바로 활용할 수 있어 초기 개발 속도가 빠르다.
- 이 앱의 핵심은 지도, 카드 탐색, 상세 정보, 즐겨찾기, 로그인 흐름이다.
- 복잡한 커스텀 렌더링보다 데이터 탐색 UX와 운영 속도가 더 중요하다.
- 어드민이 Next.js이므로 프론트엔드 사고방식과 코드 스타일을 맞추기 쉽다.

## 현재 기준 권장 조합

- App: React Native + Expo
- Map: Naver Map React Native 래퍼 검토
- Backend: Supabase
- Admin: Next.js

## Flutter를 보류한 이유

- Flutter 자체 장점은 분명하지만, 현재 프로젝트에서는 학습 비용 대비 이점이 상대적으로 작다.
- MVP 단계에서는 기술 우위보다 구현 속도와 반복 속도가 더 중요하다.

## 구현 원칙

- 모바일 앱은 Expo 중심으로 시작하되, 네이버 지도 연동 호환성을 먼저 검증한다.
- 지도 기능이 안정적으로 붙는 것을 P0로 둔다.
- 홈, 지도, 상세, 즐겨찾기 순으로 구현한다.
- 디자인은 정보형 구조 위에 감성적인 카드 톤을 입힌다.
